By the end of this guide, you'll have mastered the SaaS Boilerplate design system, enabling you to build consistent, accessible, and beautiful user interfaces with Tailwind CSS, shadcn UI components, and proper theming.

## Overview

The SaaS Boilerplate includes a comprehensive design system built on modern web standards, featuring Tailwind CSS for styling, shadcn UI for component primitives, Radix UI for accessibility, and a robust theming system. Key features include:

- **Tailwind CSS**: Utility-first CSS framework with custom design tokens
- **shadcn UI**: High-quality React components built on Radix UI primitives
- **Design Tokens**: Consistent colors, typography, spacing, and shadows
- **Dark Mode**: Complete dark/light theme support with system preference detection
- **Accessibility**: WCAG-compliant components with proper ARIA attributes
- **Responsive Design**: Mobile-first approach with breakpoint system
- **Animation System**: Smooth transitions and micro-interactions
- **Type Safety**: Full TypeScript support with proper component APIs
- **Performance**: Optimized bundle size and runtime performance

The system ensures design consistency across your application while providing flexibility for customization and extension.

### CSS Variable Updates

Dynamic theme color updates for mobile browsers:

```typescript
// Theme color script in theme provider
const THEME_COLOR_SCRIPT = `
(function() {
  var html = document.documentElement;
  var meta = document.querySelector('meta[name="theme-color"]');
  if (!meta) {
    meta = document.createElement('meta');
    meta.setAttribute('name', 'theme-color');
    document.head.appendChild(meta);
  }
  function updateThemeColor() {
    var isDark = html.classList.contains('dark');
    meta.setAttribute('content', isDark ? DARK_THEME_COLOR : LIGHT_THEME_COLOR);
  }
  var observer = new MutationObserver(updateThemeColor);
  observer.observe(html, { attributes: true, attributeFilter: ['class'] });
  updateThemeColor();
})();
`
```

## Troubleshooting

<Accordions type="multiple">
<Accordion title="Components not styling correctly">
Ensure Tailwind CSS is properly configured and the content paths include your component files. Check that the `cn` utility function is imported and used correctly for class merging.
</Accordion>

<Accordion title="Dark mode not working">
Verify that the ThemeProvider is wrapping your app and that components use CSS variables instead of hardcoded colors. Check that the `dark:` prefix is used in Tailwind classes.
</Accordion>

<Accordion title="TypeScript errors with component props">
Ensure you're extending the correct interfaces from Radix UI and including all required props. Use the `VariantProps` type from `class-variance-authority` for variant types.
</Accordion>

<Accordion title="Animations not working">
Check that `tailwindcss-animate` plugin is installed and included in the Tailwind config. Ensure animation classes are properly defined in the keyframes section.
</Accordion>

<Accordion title="Accessibility warnings">
Run accessibility audits and ensure all interactive elements have proper ARIA attributes. Use Radix UI primitives which include built-in accessibility features.
</Accordion>

<Accordion title="Bundle size concerns">
Use dynamic imports for heavy components and ensure tree shaking is working. Monitor bundle size with build tools and optimize component imports.
</Accordion>

<Accordion title="Responsive design issues">
Test components at different breakpoints. Use Tailwind's responsive prefixes (`sm:`, `md:`, `lg:`) and ensure mobile-first approach. Check container queries if needed.
</Accordion>
</Accordions>

## Best Practices

<Accordions type="multiple">
<Accordion title="Use design tokens consistently">
Always use CSS custom properties and Tailwind design tokens instead of hardcoded values. This ensures consistency and easy theme updates.
</Accordion>

<Accordion title="Follow component composition patterns">
Build complex UIs by composing smaller, reusable components. Prefer composition over inheritance for flexibility.
</Accordion>

<Accordion title="Implement proper TypeScript types">
Use strict TypeScript types for all component props. Leverage `VariantProps` for styling variants and ensure proper generic constraints.
</Accordion>

<Accordion title="Prioritize accessibility">
Include proper ARIA attributes, keyboard navigation, and screen reader support. Use Radix UI primitives for built-in accessibility.

<Accordion title="Optimize for performance">
Use React.memo for expensive components, implement proper key props for lists, and avoid unnecessary re-renders. Monitor bundle size.

<Accordion title="Test across themes and devices">

<Accordion title="Document component APIs">

<Accordion title="Use semantic HTML">
