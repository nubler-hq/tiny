By the end of this guide, you'll have a complete understanding of how to configure each MCP server in your SaaS Boilerplate project, what tools they provide, and how to use them effectively with Lia and other code agents.

## Overview

Model Context Protocol (MCP) servers extend your code agent's capabilities by providing specialized tools for different tasks. The SaaS Boilerplate configures several key MCP servers that work seamlessly with Igniter.js and your development workflow:

- **Igniter.js MCP**: All these capabilities—development lifecycle management, code generation, API validation, documentation, code analysis, project management, knowledge base, task organization, and agent delegation—are provided as part of the Igniter.js MCP server.
- **Shadcn UI MCP**: UI component scaffolding and design system integration
- **Playwright MCP**: Automated browser testing and interaction
- **Postgres MCP**: Database introspection and query operations

These servers are defined in your project's `.mcp.json` file and are automatically available to configured code agents like Lia.

## Configuration File Structure

The `.mcp.json` file in your project root defines how each MCP server runs. Here's the complete configuration structure:

```json
{
  "mcpServers": {
    "igniter": {
      "command": "npx",
      "args": ["-y", "@igniter-js/mcp-server@latest"],
      "env": {}
    },
    "browser": {
      "command": "npx",
      "args": ["@playwright/mcp@latest"],
      "env": {}
    },
    "shadcn": {
      "command": "npx",
      "args": ["shadcn@latest", "mcp"],
      "env": {}
    },
    "postgres": {
      "command": "npx",
      "args": ["-y", "@modelcontextprotocol/server-postgres", "postgresql://..."],
      "env": {
        "POSTGRES_CONNECTION_STRING": "postgresql://username:password@localhost:5432/database"
      }
    }
  }
}
```

<Accordions type="multiple">
<Accordion title="Environment Variables">
Keep sensitive information like database credentials and API tokens out of the config file. Use environment variables in the `env` section, which will be loaded from your system's environment or `.env` files.
</Accordion>
<Accordion title="Automatic Installation">
The `-y` flag in npm commands automatically installs packages if they're not present, ensuring the servers work immediately after configuration.
</Accordion>
</Accordions>

## Best Practices

<Accordions type="multiple">
<Accordion title="Security Considerations">
- Never hardcode database credentials in `.mcp.json`
- Use environment variables for sensitive configuration
- Limit database permissions to read-only where possible for development
</Accordion>
<Accordion title="Performance Optimization">
- Configure appropriate timeouts for long-running operations
- Use connection pooling for high-traffic scenarios
- Cache frequently accessed schema information
</Accordion>
<Accordion title="Error Handling">
- Implement retry logic for transient failures
- Log errors for debugging while avoiding sensitive data exposure
- Provide meaningful error messages to users
</Accordion>
<Accordion title="Integration with Lia">
- Lia automatically discovers and uses available MCP tools
- Request permission before executing destructive operations
- Combine multiple tools for complex workflows (e.g., generate code + run tests)
</Accordion>
</Accordions>

## Troubleshooting

<Accordions type="multiple">
<Accordion title="Server Won't Start">
- Check that all required packages are installed
- Verify environment variables are set correctly
- Ensure network connectivity for external services
</Accordion>
<Accordion title="Tool Execution Fails">
- Validate input parameters match tool specifications
- Check for permission issues (database, file system)
- Review error logs for detailed failure information
</Accordion>
<Accordion title="Performance Issues">
- Adjust timeout values for slow operations
- Optimize queries and reduce data transfer
- Consider caching for frequently accessed data
</Accordion>
</Accordions>
